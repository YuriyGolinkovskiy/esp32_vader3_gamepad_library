# Vader3 Gamepad Library

Библиотека для подключения геймпада **Flydigi Vader 3 Pro** к ESP32 по Bluetooth Low Energy (BLE).

## Особенности
- 🎮 Полная поддержка всех кнопок, стиков и триггеров.
- ⚡ Автоматическое переподключение при обрыве связи.
- 🛠 Удобный API: `justPressed()`, `getLeftX()`, `getDPad()` и т.д.
- 📦 Не требует знания низкоуровневых BLE-протоколов.
- 🔄 Защита от повторной инициализации и корректное освобождение ресурсов.

## Установка
1. Скачайте архив с библиотекой или клонируйте репозиторий.
2. В Arduino IDE выберите: **Скетч -> Подключить библиотеку -> Добавить .ZIP библиотеку**.
3. Убедитесь, что у вас установлены пакеты для **ESP32**.

## Быстрый старт

```cpp
#include <Vader3Gamepad.h>

Vader3Gamepad gamepad;

void setup() {
  Serial.begin(115200);
  // begin() вернет false, если библиотека уже инициализирована
  if (!gamepad.begin()) {
    Serial.println("Ошибка: библиотека уже инициализирована!");
  }
}

void loop() {
  gamepad.update(); // Обязательный вызов в цикле

  if (gamepad.isConnected()) {
    if (gamepad.justPressed(BTN_A)) {
      Serial.println("Кнопка A нажата!");
    }

    int lx = gamepad.getLeftX();
    int ly = gamepad.getLeftY();
    // ...
  }
}
```

## API Reference

### Инициализация
| Метод | Описание |
| :--- | :--- |
| `bool begin(int scanSeconds = 5)` | Запуск поиска геймпада. Возвращает `false` при повторном вызове без `end()`. |
| `void end()` | Отключение от геймпада и освобождение ресурсов. Позволяет вызвать `begin()` заново. |
| `void update()` | Обновление состояния кнопок и осей. Вызывать в `loop()`. |
| `bool isConnected()` | Возвращает `true`, если геймпад подключен. |

### Кнопки
| Метод | Описание |
| :--- | :--- |
| `bool pressed(VaderButton btn)` | `true`, если кнопка удерживается. |
| `bool justPressed(VaderButton btn)` | `true` в момент нажатия (один кадр). |
| `bool justReleased(VaderButton btn)` | `true` в момент отпускания (один кадр). |

### Стики и триггеры (Analog)
| Метод | Диапазон | Описание |
| :--- | :--- | :--- |
| `getLeftX()` / `getRightX()` | -127 ... 127 | Горизонтальная ось стика. |
| `getLeftY()` / `getRightY()` | -127 ... 127 | Вертикальная ось стика. |
| `getLeftTrigger()` / `getRightTrigger()` | 0 ... 255 | Степень нажатия триггера. |
| `getLeftXraw()` ... | 0 ... 255 | "Сырые" данные с датчиков. |

### D-Pad (Крестовина)
| Метод | Описание |
| :--- | :--- |
| `getDPad()` | Возвращает текущее направление (`DPAD_UP`, `DPAD_DOWN_LEFT` и т.д.). |
| `dpadPressed(DPadDirection dir)` | `true`, если крестовина направлена в указанную сторону. |
| `dpadJustPressed(DPadDirection dir)` | `true` в момент изменения направления. |
| `getDPadX()` / `getDPadY()` | Возвращает -1, 0 или 1 для удобства движения персонажа. |

## Поддерживаемые кнопки
| Имя в коде | Кнопка на геймпаде |
| :--- | :--- |
| `BTN_A`, `BTN_B`, `BTN_X`, `BTN_Y` | Основные кнопки |
| `BTN_LB`, `BTN_RB` | Бамперы |
| `BTN_LT`, `BTN_RT` | Триггеры (аналоговые) |
| `BTN_START`, `BTN_BACK` | Сервисные кнопки |
| `BTN_HOME` | Кнопка "Домой" |
| `BTN_C1` ... `BTN_C6` | Дополнительные макро-кнопки |

## Архитектура
Библиотека использует стандартный HID-профиль (`0x1812`). Геймпад выступает в роли сервера, а ESP32 — клиента.

> **Важно:** Для стабильной работы не забывайте вызывать `gamepad.update()` как можно чаще.
